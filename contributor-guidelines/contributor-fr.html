Lignes directrices pour les contributeurs

Tout le monde peut apporter sa contribution au développement de la Boîte à outils de l’expérience Web (BOEW). Plus les contributeurs sont variés, meilleurs sont les résultats.

La Boîte à outils de l’expérience Web est gérée ouvertement sur GitHub, ce qui permet de tenir des discussions et d’encourager le libre partage d’idées. Publiez un commentaire, signalez un bogue ou suggérez d’aborder un problème de fonctionnalité. Chaque répertoire dispose d’un système de suivi des problèmes. Utilisez celui qui correspond à la base de code avec laquelle vous travaillez. Si vous ne savez pas très bien duquel il s’agit, publiez votre commentaire sous le répertoire «WET-BOEW» à partir de GitHub.

Un processus d’évaluation des contributions à plusieurs niveaux permet d’assurer l’intégrité du code. Une fois le projet mis sur pied, des tests automatiques sont réalisés au moyen de Travis CI. Ensuite, des évaluations manuelles sont réalisées par des experts préalablement à l’approbation du nouveau code.

Looking for WET v3.1?

WET v4.0 is now the current version. There is a new location for the current page in WET v3.1.

Voici les domaines dans lesquels nous avons besoin de contributeurs :

  • Publication des demandes

    Demander d’apporter des modifications au code.

  • Problèmes et suggestions

    Signaler un bogue ou suggérer une fonctionnalité ou une amélioration.

  • Documentation

    Préparer de la documentation sur la mise en œuvre de modules d’extension ou de correctifs.

  • Évaluations heuristiques sur la facilité d’emploi

    Mener des évaluations heuristiques sur un composant de la BOEW.

  • Guides sur le Web

    Suggérer d’apporter des corrections ou des améliorations aux lignes directrices.

  • Test

    Comment tester votre code avant de l’appliquer au projet.
